## ВВЕДЕНИЕ

В современном мире веб-технологии играют ключевую роль в предоставлении пользователям доступа к различным сервисам и контенту. Одной из наиболее востребованных областей является разработка веб-приложений для работы с мультимедийным контентом, в частности, для прослушивания музыки. 

Актуальность данной работы обусловлена растущей популярностью стриминговых сервисов и необходимостью создания удобных и функциональных веб-приложений для работы с музыкальным контентом. Разработка собственного веб-приложения позволяет получить практический опыт в области full-stack разработки, освоить современные технологии и методы проектирования веб-систем.

Целью данной курсовой работы является разработка полнофункционального веб-приложения для прослушивания музыки на основе Yandex Music API, которое обеспечивает поиск треков, воспроизведение музыки, управление плейлистами и другие функции современного музыкального сервиса.

Для достижения поставленной цели необходимо решить следующие задачи:
1. Провести анализ современных технологий веб-разработки и выбрать оптимальный технологический стек
2. Спроектировать архитектуру веб-приложения с учетом требований функциональности и масштабируемости
3. Реализовать backend-часть приложения с использованием Flask
4. Реализовать frontend-часть приложения с использованием HTML, CSS и JavaScript
5. Интегрировать приложение с Yandex Music API для получения музыкального контента
6. Реализовать систему хранения данных пользователей (избранное, история, плейлисты)
7. Обеспечить адаптивность интерфейса для различных устройств
8. Провести тестирование функциональности приложения

Объектом исследования является процесс разработки веб-приложений для работы с мультимедийным контентом.

Предметом исследования являются методы и технологии создания клиент-серверных веб-приложений с интеграцией внешних API.

Практическая значимость работы заключается в создании рабочего веб-приложения, демонстрирующего применение современных технологий веб-разработки и методов интеграции с внешними сервисами. Разработанное приложение может быть использовано в качестве основы для более сложных музыкальных сервисов или как учебный пример для изучения full-stack разработки.

## 1. ОБЗОР ТЕХНОЛОГИЙ И МЕТОДОВ РАЗРАБОТКИ ВЕБ-ПРИЛОЖЕНИЙ

### 1.1. Современные технологии веб-разработки

Современная веб-разработка представляет собой комплексную область, включающую множество технологий и инструментов. Основными компонентами веб-приложения являются клиентская часть (frontend) и серверная часть (backend), которые взаимодействуют через протокол HTTP/HTTPS.

Клиентская часть веб-приложения отвечает за отображение пользовательского интерфейса и взаимодействие с пользователем. Основными технологиями для frontend-разработки являются HTML (HyperText Markup Language) для структурирования контента, CSS (Cascading Style Sheets) для стилизации и JavaScript для реализации интерактивности.

HTML5 предоставляет современные возможности для работы с мультимедийным контентом, включая элемент `<audio>` для воспроизведения аудио без использования плагинов. Это особенно важно для музыкальных приложений, так как позволяет реализовать встроенный аудиоплеер непосредственно в браузере.

CSS3 расширяет возможности стилизации, предоставляя градиенты, анимации, трансформации и другие визуальные эффекты. Современные CSS-фреймворки, такие как Bootstrap, Material UI, Tailwind CSS, упрощают создание адаптивных и визуально привлекательных интерфейсов.

JavaScript является основным языком программирования для клиентской части. Современный JavaScript (ES6+) поддерживает асинхронное программирование через Promises и async/await, что критично для работы с API и обработки асинхронных операций, таких как загрузка музыкальных треков.

Серверная часть веб-приложения отвечает за обработку запросов, работу с базами данных, интеграцию с внешними API и бизнес-логику. Для backend-разработки используются различные языки программирования и фреймворки.

Python является одним из наиболее популярных языков для веб-разработки благодаря своей читаемости, обширной экосистеме библиотек и активному сообществу разработчиков. Flask представляет собой легковесный и гибкий веб-фреймворк для Python, который позволяет быстро создавать веб-приложения с минимальными накладными расходами.

Flask предоставляет механизм маршрутизации (routing) для определения URL-эндпоинтов, систему шаблонов для генерации HTML, поддержку сессий и cookies для управления состоянием пользователя, а также возможность создания RESTful API для взаимодействия с клиентской частью.

### 1.2. Архитектура клиент-серверных приложений

Клиент-серверная архитектура является основой современных веб-приложений. В данной архитектуре клиент (веб-браузер) отправляет HTTP-запросы на сервер, который обрабатывает их и возвращает ответы, содержащие данные или HTML-разметку.

REST (Representational State Transfer) представляет собой архитектурный стиль для проектирования веб-сервисов. RESTful API использует стандартные HTTP-методы (GET, POST, PUT, DELETE) для выполнения операций с ресурсами, что обеспечивает простоту и понятность взаимодействия между клиентом и сервером.

В разрабатываемом приложении используется RESTful подход для организации API. Например, эндпоинт `GET /api/search?q=<query>` используется для поиска музыки, `GET /api/track/<track_id>` — для получения информации о треке, `POST /api/auth/login` — для аутентификации пользователя.

Асинхронная обработка запросов является важным аспектом современной веб-разработки. JavaScript на клиентской стороне использует Fetch API или XMLHttpRequest для выполнения асинхронных HTTP-запросов, что позволяет обновлять контент страницы без её полной перезагрузки (технология AJAX).

### 1.3. RESTful API и работа с внешними API

RESTful API обеспечивает стандартизированный способ взаимодействия между различными компонентами приложения. В контексте разрабатываемого веб-приложения, RESTful API используется как для внутренней коммуникации между frontend и backend, так и для интеграции с внешним сервисом Yandex Music API.

Yandex Music API представляет собой неофициальный API, полученный путем обратной разработки (reverse engineering) официального API сервиса Яндекс.Музыка. Библиотека `yandex-music-api` для Python предоставляет удобный интерфейс для работы с этим API, абстрагируя сложность низкоуровневых HTTP-запросов.

Работа с внешними API требует обработки различных сценариев: успешные запросы, ошибки сети, таймауты, ограничения скорости запросов (rate limiting). В разрабатываемом приложении реализована обработка ошибок через try-except блоки и возврат понятных сообщений об ошибках клиентской части.

Кэширование данных является важным аспектом оптимизации работы с внешними API. В приложении реализовано кэширование информации о треках, альбомах и исполнителях в локальной базе данных SQLite, что уменьшает количество запросов к внешнему API и повышает скорость работы приложения.

### 1.4. Фреймворки для backend-разработки

Выбор фреймворка для backend-разработки зависит от требований проекта, масштаба приложения и предпочтений разработчика. Flask был выбран для данного проекта по следующим причинам:

1. **Легковесность**: Flask имеет минимальный набор зависимостей и не навязывает определенную структуру проекта, что позволяет гибко организовать код
2. **Простота использования**: Flask имеет интуитивно понятный API и минимальную кривую обучения
3. **Расширяемость**: Flask поддерживает множество расширений для различных задач (аутентификация, работа с базами данных, CORS и т.д.)
4. **Подходит для небольших и средних проектов**: Для курсовой работы Flask является оптимальным выбором, так как позволяет быстро реализовать необходимый функционал без излишней сложности

Flask использует декораторы для определения маршрутов. Например, `@app.route('/api/search', methods=['GET'])` создает эндпоинт для обработки GET-запросов по пути `/api/search`. Функция, декорированная таким образом, обрабатывает входящие запросы и возвращает ответы в формате JSON через `jsonify()`.

Управление сессиями в Flask позволяет хранить данные пользователя между запросами. В разрабатываемом приложении сессии используются для хранения токена авторизации Yandex Music API, что позволяет пользователю не вводить токен при каждом запросе.

### 1.5. Инструменты для frontend-разработки

Frontend-разработка требует использования различных инструментов для создания современного и функционального пользовательского интерфейса. В данном проекте используется подход "vanilla" JavaScript без фреймворков, что позволяет лучше понять основы веб-разработки.

HTML5 предоставляет семантические элементы для структурирования контента: `<header>`, `<main>`, `<aside>`, `<footer>`, что улучшает доступность и SEO-оптимизацию страницы. Элемент `<audio>` используется для воспроизведения музыкальных треков с поддержкой управления через JavaScript API.

CSS3 обеспечивает современную стилизацию с использованием:
- CSS Variables (кастомные свойства) для централизованного управления цветовой схемой и размерами
- Flexbox и Grid Layout для создания адаптивных макетов
- Анимации и переходы для улучшения пользовательского опыта
- Media queries для адаптивности под различные размеры экранов

JavaScript обеспечивает интерактивность приложения. В проекте используется модульный подход к организации кода с разделением на функции для различных задач: работа с API, управление состоянием плеера, обработка событий пользователя.

DOM (Document Object Model) API позволяет динамически изменять содержимое страницы без её перезагрузки. Это используется для отображения результатов поиска, обновления информации о текущем треке, управления плейлистами.

Event-driven архитектура JavaScript позволяет создавать отзывчивые интерфейсы. Обработчики событий (event listeners) реагируют на действия пользователя (клики, ввод текста, изменения состояния элементов) и инициируют соответствующие операции.

### 1.6. Базы данных и системы хранения данных

База данных является критически важным компонентом веб-приложения для хранения пользовательских данных, кэша и другой информации. В разрабатываемом приложении используется SQLite — легковесная реляционная база данных, которая хранится в виде файла.

SQLite была выбрана по следующим причинам:
1. **Простота развертывания**: Не требует отдельного сервера базы данных, данные хранятся в файле
2. **Достаточная функциональность**: Поддерживает SQL, транзакции, внешние ключи, что достаточно для задач курсовой работы
3. **Нулевая конфигурация**: Не требует настройки и администрирования
4. **Хорошая производительность**: Для небольших и средних объемов данных SQLite показывает отличную производительность

Структура базы данных включает следующие таблицы:
- `users` — хранение информации о пользователях (логин, хеш пароля, токен)
- `favorite_tracks` — избранные треки пользователей
- `play_history` — история прослушиваний
- `search_history` — история поисковых запросов
- `user_playlists` — пользовательские плейлисты
- `playlist_tracks` — треки в плейлистах
- `track_cache` — кэш информации о треках

Реляционная модель данных обеспечивает целостность данных через внешние ключи и позволяет эффективно выполнять запросы с использованием JOIN операций. Например, для получения плейлиста со всеми треками используется JOIN между таблицами `user_playlists` и `playlist_tracks`.

Миграции базы данных реализованы программно через проверку существования таблиц и колонок. При инициализации приложения выполняется проверка структуры базы данных и при необходимости создаются недостающие таблицы или добавляются новые колонки, что обеспечивает совместимость с предыдущими версиями.

## 2. ПРОЕКТИРОВАНИЕ ВЕБ-ПРИЛОЖЕНИЯ

### 2.1. Анализ требований и постановка задачи

На этапе анализа требований были определены основные функциональные возможности, которые должно предоставлять веб-приложение для прослушивания музыки:

**Функциональные требования:**
1. Поиск музыкального контента (треки, альбомы, исполнители, подкасты)
2. Воспроизведение музыкальных треков через встроенный аудиоплеер
3. Управление воспроизведением (play/pause, переключение треков, регулировка громкости)
4. Просмотр чарта популярной музыки
5. Создание и управление пользовательскими плейлистами
6. Добавление треков в избранное
7. Просмотр истории прослушиваний
8. Система аутентификации пользователей
9. Хранение и управление токеном Yandex Music API
10. Полноэкранный режим воспроизведения с расширенными возможностями

**Нефункциональные требования:**
1. Адаптивный дизайн для различных устройств (десктоп, планшет, мобильный)
2. Современный и привлекательный пользовательский интерфейс
3. Быстрая загрузка и отзывчивость интерфейса
4. Обработка ошибок и информативные сообщения для пользователя
5. Безопасное хранение паролей (хеширование)
6. Кэширование данных для оптимизации производительности

**Ограничения:**
1. Работа с неофициальным API, что может привести к изменениям в работе при изменении API Яндекс.Музыка
2. Без токена доступны только первые 30 секунд треков
3. Прямые ссылки на треки имеют ограниченное время жизни (около 1 минуты)
4. Зависимость от доступности внешнего API

### 2.2. Выбор технологического стека

На основе анализа требований был выбран следующий технологический стек:

**Backend:**
- **Python 3.7+** — язык программирования
- **Flask** — веб-фреймворк для создания API и обработки запросов
- **yandex-music-api** — библиотека для работы с Yandex Music API
- **SQLite** — база данных для хранения пользовательских данных
- **Werkzeug** — библиотека для хеширования паролей (входит в Flask)

**Frontend:**
- **HTML5** — разметка страницы
- **CSS3** — стилизация с использованием современных возможностей (CSS Variables, Flexbox, Grid, Animations)
- **Vanilla JavaScript (ES6+)** — логика клиентской части без использования фреймворков

**Инструменты разработки:**
- **Git** — система контроля версий
- **Visual Studio Code / PyCharm** — интегрированная среда разработки

Выбор данного стека обоснован следующими факторами:
1. Python и Flask обеспечивают быструю разработку и простоту поддержки кода
2. Vanilla JavaScript позволяет лучше понять основы веб-разработки без абстракций фреймворков
3. SQLite не требует дополнительной настройки и подходит для учебного проекта
4. Все технологии имеют хорошую документацию и активное сообщество

### 2.3. Проектирование архитектуры приложения

Архитектура приложения построена по принципу разделения на слои:

**Слой представления (Presentation Layer):**
- HTML-шаблоны для отображения интерфейса
- CSS для стилизации
- JavaScript для интерактивности и взаимодействия с API

**Слой приложения (Application Layer):**
- Flask-роуты для обработки HTTP-запросов
- Бизнес-логика обработки данных
- Валидация входных данных
- Формирование ответов в формате JSON

**Слой данных (Data Layer):**
- Модуль `database.py` для работы с SQLite
- Модуль `database_api.py` для API-эндпоинтов работы с БД
- Кэширование данных треков, альбомов, исполнителей

```python
                    'artists': [{'id': a.id, 'name': a.name} for a in track_short.artists],
                    'cover_uri': get_cover_url(track_short.cover_uri),
                    'duration_ms': track_short.duration_ms
                }
                result['tracks'].append(track_data)
        
        # Аналогично для альбомов, исполнителей, подкастов
        
        # Сохранение в историю поиска
        total_results = len(result['tracks']) + len(result['albums']) + \
                       len(result['artists']) + len(result['podcasts'])
        db.add_search_history(query, 'all', total_results, session.get('user_id') or 0)
        
        return jsonify(result)
    except YandexMusicError as e:
        return jsonify({'error': str(e)}), 500
```

```javascript
        let nextIndex = Math.floor(Math.random() * currentPlaylist.length);
        while (nextIndex === currentTrackIndex && currentPlaylist.length > 1) {
            nextIndex = Math.floor(Math.random() * currentPlaylist.length);
        }
        currentTrackIndex = nextIndex;
        playTrack(currentPlaylist[nextIndex].id);
        return;
    }
    
    if (currentTrackIndex < currentPlaylist.length - 1) {
        currentTrackIndex++;
        playTrack(currentPlaylist[currentTrackIndex].id);
    } else if (repeatMode === 'all' && isAuto) {
        currentTrackIndex = 0;
```

```python
def add_play_history(self, track_data, play_duration=0, user_id=0):
    # Удаление старой записи с таким же track_id (перемещение вверх)
    cursor.execute('DELETE FROM play_history WHERE track_id = ? AND user_id = ?',

        (user_id, track_id, title, artists, cover_uri, duration_ms, play_duration)
        VALUES (?, ?, ?, ?, ?, ?, ?)
```

### 5.5. Обработка ошибок и исключений

Обработка ошибок реализована на всех уровнях приложения:

**Backend:**
```python
try:
    # Работа с API
    search_result = client.search(query)
except YandexMusicError as e:
    logger.error(f"Ошибка Yandex Music API: {e}")
    return jsonify({'error': str(e)}), 500
except Exception as e:
    logger.error(f"Неожиданная ошибка: {e}")
    import traceback
    logger.error(traceback.format_exc())
    return jsonify({'error': 'Внутренняя ошибка сервера'}), 500
```

**Frontend:**
```javascript
try {
    const response = await fetch(url);
    if (!response.ok) {
        const data = await response.json();
        showError(data.error || 'Ошибка запроса');
        return;
    }
    const data = await response.json();
    // Обработка успешного ответа
} catch (error) {
    console.error('Ошибка:', error);
    showError('Ошибка соединения с сервером');
}
```

**Пользовательские сообщения:**
- Понятные сообщения об ошибках на русском языке

### Приложение В. Схема базы данных

**Таблица users:**
- id (PRIMARY KEY)
- username (UNIQUE)
- password_hash
- token
- created_at

**Таблица favorite_tracks:**
- id (PRIMARY KEY)
- user_id (FOREIGN KEY → users.id)
- track_id
- title, artists, duration_ms, cover_uri
- added_at
- UNIQUE(user_id, track_id)

**Таблица play_history:**
- id (PRIMARY KEY)
- user_id (FOREIGN KEY → users.id)
- track_id, title, artists
- cover_uri, duration_ms
- played_at, play_duration

**Таблица user_playlists:**
- id (PRIMARY KEY)
- user_id (FOREIGN KEY → users.id)
- name, description, cover_uri
- created_at, updated_at

**Таблица playlist_tracks:**
- id (PRIMARY KEY)
- playlist_id (FOREIGN KEY → user_playlists.id)
- track_id, position
- added_at
- UNIQUE(playlist_id, track_id)

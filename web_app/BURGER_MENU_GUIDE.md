## 🎨 Визуальные элементы

### 1. Кнопка бургер-меню
```
[☰]  →  [×]
 |       |
Закрыто  Открыто
```

**Характеристики:**
- Размер: 44×44px (touch-friendly)
- Анимация: 3 линии → крестик
- Цвет: полупрозрачный белый
- Hover: подсветка оранжевым

### 2. Боковая панель
- Ширина: min(320px, 85vw)
- Анимация: выезжает слева
- Фон: glassmorphism с blur
- Высота: 100vh (на весь экран)

### 3. Overlay (затемнение)
- Фон: rgba(0, 0, 0, 0.6) + blur
- Закрывает меню при клике
- Плавное появление/исчезновение

### 4. Шапка меню (только mobile)
```
┌─────────────────────────┐
│ Навигация           [×] │
├─────────────────────────┤
│ • Главная               │
│ • Чарт                  │
│ ...                     │
```

## 🔧 Функциональность

### Открытие меню:
1. Клик на кнопку бургер `[☰]`
2. Sidebar выезжает слева
3. Overlay затемняет фон
4. Body блокирует скролл
5. Фокус переходит на кнопку закрытия

### Закрытие меню:
1. **Клик на [×]** в шапке меню
2. **Клик на overlay** (затемнённую область)
3. **Клик на пункт меню** (автоматически)
4. **ESC на клавиатуре**
5. **Изменение размера на desktop** (авто-закрытие)

### Анимации:
- Бургер → Крестик: **0.3s**
- Sidebar выезд: **0.3s cubic-bezier**
- Overlay fade: **0.3s**

## 🐛 Troubleshooting

### Проблема: Меню не открывается
**Решение:**
1. Проверьте консоль браузера (F12) на ошибки
2. Убедитесь, что `burger-menu.js` загружается
3. Проверьте, что элементы имеют правильные ID

### Проблема: Меню не закрывается при клике на пункт
**Решение:**
Убедитесь, что у пунктов меню есть класс `.nav-item`:
```javascript
// В burger-menu.js проверяется класс nav-item
navItems = document.querySelectorAll('.nav-item');
```

### Проблема: Анимация дёргается
**Решение:**
Добавьте `will-change` для оптимизации:
```css
.sidebar {
    will-change: left;
}
```

### Проблема: Бургер видно на desktop
**Решение:**
Проверьте медиазапрос в `burger-menu.css`:
```css
@media (max-width: 991px) {
    .burger-menu-btn { display: flex; }
}
```
